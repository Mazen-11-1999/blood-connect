# الميزات الجديدة - إنـقـاذ حــيـاة

## 🚨 نظام الحالات الطارئة والإشعارات

### الميزات المضافة:

#### 1. **حالة طارئة (Urgent)**
- ✅ خيار "حالة طارئة" عند إرسال الرسالة
- ✅ رقم الهاتف يصبح إلزامياً في الحالات الطارئة
- ✅ تمييز الرسائل الطارئة في قائمة الرسائل (لون أحمر، نبض)
- ✅ ترتيب الرسائل: الطارئة أولاً

#### 2. **وقت الحاجة (Needed DateTime)**
- ✅ حقل لتحديد التاريخ والوقت المطلوب للدم
- ✅ عرض وقت الحاجة بشكل بارز في الرسالة
- ✅ إظهار وقت الحاجة في الإشعارات

#### 3. **إشعارات المتصفح (Browser Notifications)**
- ✅ إشعارات فورية عند استقبال رسالة جديدة
- ✅ إشعارات خاصة للحالات الطارئة (وميض، صوت، اهتزاز)
- ✅ أزرار تفاعلية في الإشعار:
  - "اتصل الآن" - للاتصال المباشر
  - "عرض الرسالة" - لفتح الرسالة في التطبيق
- ✅ يعمل حتى عند إغلاق المتصفح (Service Worker)

#### 4. **Service Worker**
- ✅ إشعارات فورية حتى عند إغلاق المتصفح
- ✅ فحص تلقائي للرسائل الجديدة كل 30 ثانية
- ✅ تخزين مؤقت (Cache) لتحسين الأداء
- ✅ يعمل في الخلفية

#### 5. **Progressive Web App (PWA)**
- ✅ يمكن تثبيت التطبيق على الموبايل
- ✅ يعمل كتطبيق مستقل
- ✅ إشعارات فورية على الموبايل
- ✅ أيقونة على الشاشة الرئيسية

#### 6. **محاكاة SMS**
- ✅ في الحالات الطارئة، يتم محاكاة إرسال SMS
- ✅ في النسخة الحقيقية، يمكن ربطه بـ API مثل Twilio

---

## 📱 كيفية الاستخدام

### للمحتاج (إرسال رسالة طارئة):

1. ابحث عن متبرع
2. اضغط "إرسال رسالة"
3. حدد ✅ "حالة طارئة"
4. حدد وقت الحاجة (اختياري)
5. اكتب الرسالة
6. **ضع رقم هاتفك (إلزامي في الحالات الطارئة)**
7. اضغط "إرسال"

**النتيجة:**
- ✅ إشعار فوري للمتبرع (حتى لو كان المتصفح مغلق)
- ✅ محاكاة إرسال SMS
- ✅ الرسالة تظهر في أعلى القائمة

### للمتبرع (استقبال رسالة طارئة):

1. **إشعار فوري** يظهر على الشاشة (حتى لو كان المتصفح مغلق)
2. يمكنه:
   - الضغط على "اتصل الآن" للاتصال المباشر
   - الضغط على "عرض الرسالة" لفتح التطبيق
3. الرسالة تظهر في صفحة "الرسائل" مع:
   - شارة "حالة طارئة" حمراء
   - وقت الحاجة (إن وجد)
   - رقم الهاتف مع زر "اتصل الآن"

## 🔧 التقنيات المستخدمة

1. **Web Notifications API** - لإشعارات المتصفح
2. **Service Worker** - للإشعارات الفورية
3. **Progressive Web App (PWA)** - للتثبيت على الموبايل
4. **LocalStorage** - لتخزين البيانات (يمكن استبداله بقاعدة بيانات)

---

## 🚀 التطوير المستقبلي

### SMS حقيقي:
```javascript
// يمكن إضافة API حقيقي مثل Twilio
async function sendSMS(phone, message) {
    const response = await fetch('https://api.twilio.com/...', {
        method: 'POST',
        headers: {
            'Authorization': 'Basic ...',
            'Content-Type': 'application/x-www-form-urlencoded'
        },
        body: `To=${phone}&Message=${message}`
    });
    return response.json();
}
```

### Push Notifications:
- يمكن إضافة Firebase Cloud Messaging (FCM)
- إشعارات فورية على جميع الأجهزة

### قاعدة بيانات:
- استبدال LocalStorage بـ Firebase أو MongoDB
- تخزين آمن للبيانات

## 📝 ملاحظات مهمة

1. **الإشعارات تحتاج إذن المستخدم** - سيتم طلب الإذن عند أول استخدام
2. **Service Worker يحتاج HTTPS** - في الإنتاج (يمكن استخدام localhost للتطوير)
